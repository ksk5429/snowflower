# Title formulas (snowflower)

Working hypotheses, refine after first 10 published pieces.

## Hooks that work for accessibility content (per channels we've studied)

| Pattern | Example | Where it lands |
|---|---|---|
| Counterfactual | "What if you couldn't use your hands?" | YouTube long-form |
| Before/After | "I used my phone for a week without touching it" | TikTok, Reels |
| Tier list / ranking | "The 5 hands-free input tools, ranked" | YouTube + LinkedIn |
| Tool head-to-head | "MouthPad vs Quha vs Glassouse — what each one is actually for" | YouTube long-form |
| Quiet expertise | "How OTs choose between voice and switch input" | LinkedIn, newsletter |
| User profile | "How [name] reads, writes, codes — without touching a keyboard" | YouTube + IG Reels |

## Anti-patterns (skip)

- Clickbait that overpromises ("This will change EVERYTHING")
- Pity framing ("Inspirational disabled person uses TECH!!!")
- Curing/fixing language — disability is not a problem to solve
- "Hack" / "secret" / "trick" — undermines accessibility-tech credibility
- Numbers without specificity ("X% better") without source

## Disclosure-friendly titles (for sponsored episodes later)

- "Sponsored deep-dive: how augmental built MouthPad^"  ← clear paid
- "What we learned testing the Quha Zono 2 (loaner)"   ← sample-disclosed
- Never disguise sponsored as editorial. KFTC + FTC both fine for this.
